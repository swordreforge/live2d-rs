# live2d-rs

Rust bindings and viewer for [Live2D Cubism SDK for Native](https://www.live2d.com/download/cubism-sdk/download-native/).

## Workspace Structure

| Crate | Description |
|-------|-------------|
| `live2d-core-sys` | Raw `-sys` bindings generated by `bindgen` over the Cubism Core C API (`csmMoc`, `csmModel`, etc.) |
| `live2d-core` | Safe Rust wrapper — `Moc`, `Model`, `Parameters`, `Parts`, `Drawables`, `CanvasInfo`, `OffscreenInfos` |
| `live2d-viewer` | Desktop viewer: `winit` + `glutin` + `glow` (OpenGL 3.3 Core) + `egui` UI overlay |

## Quick Start

### Prerequisites

- Rust 1.75+
- Cubism Core native library (`liblive2d.so` / `live2d.dll` / `liblive2d.dylib`)
- OpenGL 3.3 Core capable GPU

### Setup

```bash
# 1. Download Cubism SDK for Native from Live2D
#    https://www.live2d.com/download/cubism-sdk/download-native/

# 2. Place the Core library where the linker can find it:
#    Linux:   copy liblive2d.so.5 to /usr/local/lib/
#    macOS:   copy liblive2d.5.dylib to /usr/local/lib/
#    Windows: copy live2d.dll alongside the binary

# 3. Set CUBISM_CORE_DIR or point to it in live2d-core-sys/build.rs

# 4. Build & run
cargo build --release
cargo run --release -p live2d-viewer -- <path-to-model-directory>
```

### Running with sample models

Sample models are bundled in the Cubism SDK under `Samples/Resources/`:

```bash
cargo run --release -p live2d-viewer -- \
    /path/to/CubismSdkForNative-5-r.5/Samples/Resources/Haru
```

Use the GUI to:
- **Model**: Select between available model files
- **Parameters**: Adjust any parameter slider (mouth, eyes, eyebrows, body, etc.)

## Crate Details

### `live2d-core-sys` (`live2d_core_sys`)

Auto-generated FFI declarations via `bindgen`. Generates at build time from `CubismCore/include/Live2DCubismCore.h`.

Key types: `csmMoc`, `csmModel`, `csmDrawableVertexPositions`, `csmVector2`, etc.

### `live2d-core` (`live2d_core`)

Safe zero-cost wrappers with lifetimes:

```rust
let moc = Moc::from_bytes(&moc3_data)?;       // load MOC3 data
let mut model = Model::initialize(&moc)?;     // create model instance

let canvas = model.canvas_info();              // pixel size, origin
let params = model.parameters();              // parameter IDs, values, ranges
let parts = model.parts();                    // part opacity
let drawables = model.drawables();            // vertex/index data, textures, masks
let orders = model.render_orders();           // draw order

model.update();                                // run physics/motion
```

### `live2d-viewer` (`live2d_viewer`)

Desktop application with:

- **Winit** window and event loop
- **Glutin** OpenGL 3.3 Core context via `raw-window-handle`
- **Glow** typed OpenGL wrapper (renderer, shaders, textures, FBO)
- **Egui** immediate-mode GUI overlay (parameter sliders, model switching)
- **Live2D renderer**: supports multiply/screen blending, mask compositing (FBO + masked fragment shader)

## Architecture

```
┌─────────────────────┐
│   live2d-viewer     │  Desktop viewer (winit + glow + egui)
│   ┌──────────────┐  │
│   │ Renderer     │  │  OpenGL 3.3 Core shaders, FBO mask compositing
│   │ GUI (egui)   │  │  Parameter sliders, model selector
│   │ AppState     │  │  Model lifecycle, texture management
│   └──────────────┘  │
├─────────────────────┤
│   live2d-core       │  Safe Rust API — Moc, Model, Parameters, Drawables
├─────────────────────┤
│   live2d-core-sys   │  bindgen FFI — Cubism Core C API
├─────────────────────┤
│   Cubism Core       │  Native library (.so / .dll / .dylib)
│   (external)        │
└─────────────────────┘
```

## Shaders

Viewer uses 3 GLSL 330 Core shaders:

| Program | Uniforms | Purpose |
|---------|----------|---------|
| `program` | uTexture, uMultiplyColor, uScreenColor, uOpacity | Standard un-masked drawables |
| `mask_program` | uScale, uTranslate, uOpacity | Render mask shapes to FBO |
| `masked_program` | +uMaskTexture, uMaskSize | Per-pixel mask compositing via `gl_FragCoord` |

Mask rendering flow:
1. Bind offscreen FBO, draw mask geometry → mask alpha in FBO texture
2. Switch to masked program, bind FBO texture as `uMaskTexture`
3. Sample mask in fragment shader by computing UV from `gl_FragCoord` / `uMaskSize`
4. Multiply fragment alpha by mask value
